# nz-web 前端 UI 设计指南

## 设计定位

- 风格关键词：极简、淡绿色、安静、有秩序
- 视觉目标：保留后台管理系统的效率感，同时去掉默认组件库的“生硬感”
- 统一策略：先统一主题变量与布局骨架，再让页面跟随容器和组件基线收敛

## 主题基线

- 主色：`#6f9b76`，用于主要按钮、激活态、焦点态
- 页面背景：`#f4f7f2`，保持低饱和、大留白
- 内容面：`rgba(255, 255, 255, 0.92)`，保证信息区清晰
- 边框：`#d7e2d4`，优先用细边框而不是重阴影分层
- 正文色：`#2f3b31`
- 次级文本：`#5d6b60`

主题变量统一放在 `nz-web/src/styles/variables.css`，禁止在页面中零散硬编码主色、背景色、圆角和阴影。

## 字体与排版

- 正文字体使用本地中文优先字体栈：`MiSans`、`PingFang SC`、`Microsoft YaHei UI`
- 字重控制在 `400 / 500 / 600` 三档，避免滥用粗体
- 页面标题建议 `20px-28px`，模块标题建议 `16px-18px`
- 正文默认 `14px`，说明文本 `12px-13px`
- 行高保持宽松，正文建议不低于 `1.5`

## 布局规则

- 整体布局采用“浅底 + 浅侧栏 + 白色内容面”的三层结构
- 侧栏负责导航分组，不承担品牌宣传，不使用重渐变和高对比背景
- 顶栏只放页面身份信息和用户操作，避免堆叠搜索、面包屑、统计等无关内容
- 页面内容统一包裹在 `app-page` 容器内，集中处理间距、边框、圆角和阴影

## 组件规则

- 按钮：主按钮使用主题主色；次按钮优先默认样式，不额外加描边特效
- 表单：优先使用明确标签；登录、创建、编辑等高专注场景优先 `label-position="top"`
- 表格：表头弱底色，内容区白底；优先靠留白和字重区分层级
- 弹窗：圆角与卡片保持一致，避免弹窗和页面使用两套视觉语言
- 标签：状态必须“颜色 + 文案”同时存在，不能只依赖颜色表达

## 间距与形状

- 页面基础内边距：`24px`
- 卡片和弹窗默认圆角：`18px-24px`
- 输入框、按钮、菜单项高度优先落在 `40px-44px`
- 模块之间优先使用 `16px / 20px / 24px` 的固定节奏，不做随意微调

## 交互与可访问性

以下规则吸收了 2026-04-19 拉取的 Web Interface Guidelines：

- 交互控件必须有清晰焦点态，不能只在 hover 时变化
- 输入框要提供可见标签；登录类字段补全 `name` 和 `autocomplete`
- 可点击区域尽量不低于 `44px`
- 减少装饰性动画；默认兼容 `prefers-reduced-motion`
- 信息层级靠标题、留白、分组表达，不靠大面积色块堆叠

## 实施约定

- 优先改主题变量，不先写页面级临时覆盖
- 新增样式先放 `src/styles/variables.css`、`src/styles/layout.css` 或全局基础样式，确认无法复用后再落页面私有样式
- 页面需要新视觉容器时，先抽语义类名，再决定是否补 Uno 原子类
- 业务页沿用 Element Plus 时，先看能否通过 `--el-*` 变量解决，不直接覆盖深层内部类
