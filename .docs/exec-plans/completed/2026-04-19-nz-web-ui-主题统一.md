# 任务：统一 nz-web 前端 UI 风格与主题变量

## 背景与目标

当前 `nz-web` 主要依赖 Element Plus 默认视觉，缺少统一的主题变量、布局骨架和可复用设计规范。

本次目标：

- 制定一份前端 UI 设计指南
- 建立可复用的主题变量层
- 将主布局与登录页接入统一风格

## 范围与非目标

范围：

- `.docs` 中新增前端 UI 指南
- `nz-web/src/styles/` 新增主题变量文件
- 调整布局和登录页样式接入

非目标：

- 不逐页重写系统 CRUD 页面
- 不修改接口、路由和业务逻辑

## 受影响文件

- `.docs/index.md`
- `.docs/nz-web-ui-design-guide.md`
- `nz-web/src/styles/index.css`
- `nz-web/src/styles/variables.css`
- `nz-web/src/styles/base.css`
- `nz-web/src/styles/layout.css`
- `nz-web/src/layout/index.vue`
- `nz-web/src/layout/Header.vue`
- `nz-web/src/layout/Sidebar.vue`
- `nz-web/src/views/login/index.vue`

## 实施步骤

1. 读取现有布局、登录页和全局样式，确认统一风格落点
2. 明确“极简淡绿色”主题方向，沉淀颜色、边框、圆角、阴影和间距变量
3. 用布局容器统一后台页面外层骨架
4. 调整登录页，作为主题落地样板
5. 更新文档索引并验证构建

## 决策记录

- 风格采用低饱和淡绿色，而不是高对比深色主题，优先保证后台长期使用的耐看性
- 统一策略优先依赖 `--el-*` 和自定义 `--nz-*` 变量，减少页面散落覆盖
- 系统业务页先通过容器统一背景、留白和表格基线，不和当前业务改动冲突

## 验证方式

- 在 `nz-web` 执行 `npm run build`
- 手动检查登录页、侧栏、顶栏和常见表格页的视觉一致性

## 收尾结论

已完成主题变量、布局骨架和 UI 指南沉淀，`nz-web` 具备统一风格的基础能力。
后续新增页面可直接复用主题变量与容器规范，不需要再从默认组件样式重新拼装。
